# Data Model: Logger Package

## Entities

### `AuditLogEntry` (Input Interface)

The TypeScript interface used by callers to submit audit log entries. Defined in `packages/logger/src/audit/audit.logger.ts`.

**Storage:** Transient — passed to `AuditLogger.log()` and then persisted via `AuditLogRepository`.

| Field      | Type                     | Description                             | Constraints / Validation                |
| ---------- | ------------------------ | --------------------------------------- | --------------------------------------- | -------- |
| `action`   | `string`                 | Action identifier (e.g., `user.update`) | Required                                |
| `module`   | `string`                 | Module that originated the action       | Required                                |
| `targetId` | `string?`                | ID of the entity being acted upon       | Optional                                |
| `before`   | `Record<string, unknown> | null?`                                  | State before the change (JSON diff)     | Optional |
| `after`    | `Record<string, unknown> | null?`                                  | State after the change (JSON diff)      | Optional |
| `status`   | `string?`                | Outcome of the action                   | Optional, defaults to `'SUCCESS'`       |
| `userId`   | `string?`                | User who performed the action           | Optional, falls back to session context |
| `userRole` | `string?`                | Role of the user                        | Optional, falls back to session context |

---

### `AuditLog` (Database Entity)

The Prisma model persisted to PostgreSQL. Defined in the `@tempot/database` package's Prisma schema. The logger package writes to this table via `AuditLogRepository`.

**Storage:** PostgreSQL `audit_log` table, managed by Prisma.

| Field       | Type       | Description                    | Constraints / Validation          |
| ----------- | ---------- | ------------------------------ | --------------------------------- |
| `id`        | `String`   | Primary key (cuid)             | Auto-generated                    |
| `action`    | `String`   | Action identifier              | Required                          |
| `module`    | `String`   | Originating module             | Required                          |
| `targetId`  | `String?`  | Target entity ID               | Optional                          |
| `before`    | `Json?`    | Pre-change state               | Optional, stored as JSONB         |
| `after`     | `Json?`    | Post-change state              | Optional, stored as JSONB         |
| `status`    | `String`   | Outcome (`SUCCESS` / `FAILED`) | Required, defaults to `'SUCCESS'` |
| `userId`    | `String?`  | Acting user identifier         | Optional                          |
| `userRole`  | `String?`  | Acting user's role             | Optional                          |
| `createdAt` | `DateTime` | Timestamp of the log entry     | Auto-generated by Prisma          |

---

### `SENSITIVE_KEYS` (Configuration Constant)

A string array defining field names that must be redacted from all log output. Defined in `packages/logger/src/logger.config.ts`.

**Storage:** In-memory constant. No database persistence.

| Value        | Purpose                       |
| ------------ | ----------------------------- |
| `password`   | User credentials              |
| `token`      | Authentication/session tokens |
| `secret`     | API secrets, encryption keys  |
| `apiKey`     | Third-party API keys          |
| `creditCard` | Payment card numbers          |

---

### Serialized AppError (Output Structure)

The output structure produced by `appErrorSerializer` when serializing `AppError` instances for Pino log output. Defined in `packages/logger/src/technical/error.serializer.ts`.

**Storage:** Log output only (stdout/stderr). Not persisted to database.

| Field         | Type      | Description                                           | Conditions                            |
| ------------- | --------- | ----------------------------------------------------- | ------------------------------------- |
| `message`     | `string`  | Error message                                         | Always present                        |
| `code`        | `string`  | Error code (e.g., `AUTH_FAILED`)                      | Always present                        |
| `i18nKey`     | `string`  | Internationalization key (e.g., `errors.AUTH_FAILED`) | Always present                        |
| `stack`       | `string?` | Stack trace                                           | Only when `NODE_ENV !== 'production'` |
| `details`     | `object?` | Error details with PII redacted                       | Only when `err.details` exists        |
| `__redundant` | `boolean` | Indicates already-logged error (Rule XXIII)           | Only when `err.loggedAt` is set       |

**Redundant error output** (when `loggedAt` is already set):

| Field         | Type      | Description               |
| ------------- | --------- | ------------------------- |
| `message`     | `string`  | Always `'Already logged'` |
| `code`        | `string`  | Original error code       |
| `__redundant` | `boolean` | Always `true`             |

## Relationships

```
AuditLogEntry (input interface)
  └── written to ──> AuditLog (database entity via AuditLogRepository)

SENSITIVE_KEYS (config)
  ├── consumed by ──> Pino redact config (top-level field redaction)
  └── consumed by ──> appErrorSerializer / redactRecursive (nested field redaction)

SessionContext (from @tempot/session-manager)
  ├── read by ──> Pino mixin (injects userId into log entries)
  └── read by ──> AuditLogger.log() (fills userId/userRole when not explicit)
```

## Storage Mechanisms

- **PostgreSQL:** `AuditLog` entries are persisted via `AuditLogRepository` from `@tempot/database`. Uses Prisma ORM with JSONB columns for `before`/`after` diffs.
- **Log output (stdout):** Technical logs are written as structured JSON to stdout via Pino. Not persisted by the logger package — log aggregation is an infrastructure concern.
- **In-memory:** `SENSITIVE_KEYS` is a compile-time constant. Session context is resolved at runtime from `AsyncLocalStorage` via `@tempot/session-manager`.

## Data Flow

```
Caller (any service)
  └─ auditLogger.log({ action, module, before, after, ... })
       └─ AuditLogger merges session context (userId, userRole)
            └─ AuditLogRepository.create(data)
                 └─ Prisma persists to PostgreSQL audit_log table

Caller (any service)
  └─ logger.error({ err: appError }, 'message')
       ├─ Pino mixin injects userId from sessionContext
       ├─ Pino redact removes SENSITIVE_KEYS from top-level fields
       └─ appErrorSerializer processes AppError
            ├─ Checks loggedAt (Rule XXIII — no double logging)
            ├─ Redacts SENSITIVE_KEYS recursively from err.details
            └─ Conditionally includes stack trace (production check)
```
