## ما هو تمبوت؟

تمبوت (Template + Bot) هو إطار عمل شامل لبناء بوتات تيليجرام المؤسسية. يوفر جميع البنية التحتية جاهزة للاستخدام الفوري حتى يركز المطور فقط على منطق الأعمال.

## الطبقات الثلاث

يتبع تمبوت مبدأ العمارة النظيفة (Clean Architecture) مع ثلاث طبقات:

### طبقة النواة (Core Layer)

تحتوي على منطق الأعمال والوحدات والخدمات. هذه الطبقة مستقلة تمامًا عن التقنيات الخارجية.

### طبقة الخدمات (Services Layer)

تتعامل مع الموارد الخارجية: قاعدة البيانات، التخزين، الذكاء الاصطناعي. كل خدمة خارجية لها طبقة تجريد قابلة للاستبدال.

### طبقة الواجهات (Interface Layer)

نقطة التفاعل مع المستخدم النهائي: تيليجرام، الويب، لوحة التحكم.

## المبادئ المعمارية الخمسة

هذه القواعد غير قابلة للتفاوض ويتم فرضها في كل مراجعة كود:

### 1. طبقة تجريد لكل مورد خارجي

كل خدمة خارجية (AI، التخزين، الدفع) مغلفة في واجهة قابلة للاستبدال. هذا يسمح بتغيير المزود دون تعديل منطق الأعمال.

### 2. طبقة تخزين مؤقت موحدة

تستخدم `cache-manager` مع محولات Keyv بنظام متدرج: ذاكرة ثم Redis ثم قاعدة بيانات.

### 3. مصنع قوائم انتظار مركزي

جميع قوائم الانتظار تُدار عبر BullMQ من خلال مصنع مركزي في الحزمة المشتركة.

### 4. اتصال مبني على الأحداث

الوحدات لا تتواصل مباشرة. كل التواصل يتم عبر ناقل الأحداث (Event Bus) حصريًا.

### 5. تراجع سلس

كل خدمة خارجية لها سيناريو احتياطي موثق. إذا تعطل مزود AI، يتحول النظام للعمل بدونه.

## بنية Monorepo

يستخدم المشروع pnpm workspaces مع تقسيم واضح:

| المجلد      | الغرض                        |
| ----------- | ---------------------------- |
| `apps/`     | التطبيقات (bot-server، docs) |
| `packages/` | الحزم المشتركة (20 حزمة)     |
| `modules/`  | وحدات الأعمال                |
| `specs/`    | مواصفات SpecKit              |
| `docs/`     | التوثيق                      |

## الحزم الأساسية

| الحزمة            | الدور                                                         |
| ----------------- | ------------------------------------------------------------- |
| `shared`          | أدوات مشتركة، أنواع، `AppError`، مصنع التخزين المؤقت والقوائم |
| `logger`          | تسجيل منظم عبر Pino                                           |
| `event-bus`       | ناقل الأحداث (محلي، داخلي، خارجي عبر Redis)                   |
| `auth-core`       | صلاحيات CASL (RBAC)                                           |
| `database`        | PostgreSQL مع Prisma ORM                                      |
| `session-manager` | إدارة جلسات Redis                                             |
| `i18n-core`       | ترجمة عبر i18next (عربي أساسي + إنجليزي)                      |
| `module-registry` | اكتشاف وتحقق وتسجيل الوحدات                                   |
| `ai-core`         | تجريد مزودي AI عبر Vercel AI SDK                              |

## نمط النتيجة (Result Pattern)

يستخدم تمبوت مكتبة `neverthrow` بدلاً من رمي الاستثناءات. جميع الدوال العامة تعيد `Result<T, AppError>`:

```typescript
import { ok, err, type Result } from 'neverthrow';
import { AppError } from '@tempot/shared';

function divide(a: number, b: number): Result<number, AppError> {
  if (b === 0) {
    return err(AppError.validation('Cannot divide by zero'));
  }
  return ok(a / b);
}
```

## البنية القابلة للتوصيل

كل حزمة وكل ميزة يمكن تفعيلها أو تعطيلها بشكل مستقل عبر متغيرات البيئة (Toggle Guards). هذا يسمح بنشر خفيف يتضمن فقط الميزات المطلوبة.
