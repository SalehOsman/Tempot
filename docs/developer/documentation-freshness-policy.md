# Documentation Freshness Policy

## Active Documentation

Active guidance includes the root README, component READMEs, `docs/ROADMAP.md`,
`docs/developer/`, `docs/operations/`, and hand-authored pages under
`docs/product/`.

- Active claims must match current manifests, source, configuration names, and
  roadmap status.
- Hand-authored product pages must contain valid governed frontmatter.
- Package product pages are stale when package source has a newer committed
  change and the page has not been reviewed or updated.
- Modified product pages count as reviewed in the local working tree; CI uses
  committed timestamps.

Run:

```bash
pnpm docs:check
```

## Generated Reference

`docs/product/reference/` is generated by TypeDoc during the documentation
build. It is excluded from hand-authored frontmatter and freshness checks and
must not be edited manually.

## Archive Documentation

Files under `docs/archive/` preserve historical decisions and implementation
plans. They are not checked against current runtime claims. New archive entries
must clearly retain their original date or historical context and must not be
linked as current operational guidance.

Moving a stale active document to the archive is valid only when its guidance
is no longer authoritative and an active replacement or source-of-truth link
exists.
